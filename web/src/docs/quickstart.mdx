<ProseH2 id="prerequisites">Prerequisites</ProseH2>

- **Bun** 1.1 or later. Package manager and runtime for both backend and web.
- **Sui CLI** 1.73 or later, configured for testnet. Run `sui client envs` to verify.
- **Walrus CLI** 1.50 or later. Required for site deploys and blob ops.
- **Docker Desktop** 24 or later. Needed only if you run the Sui Stack Messaging relayer locally.
- **PostgreSQL** 15 or later. Backing store for the backend.

<ProseH2 id="clone-and-install">Clone and Install</ProseH2>

<CodeBlock
  lang="bash"
  code={`git clone https://github.com/your-org/lighthouse.git
cd lighthouse

cd backend && bun install
cd ../web && bun install`}
/>

<ProseH2 id="backend-environment">Backend Environment</ProseH2>

Copy `backend/.env.example` to `backend/.env`. All testnet object IDs are pre-filled — only fill in the blanked credentials below.

| Key                                         | How to get it                                            |
| :------------------------------------------ | :------------------------------------------------------- |
| `DATABASE_URL`                              | Your local PostgreSQL connection string                  |
| `JWT_SECRET`                                | `openssl rand -hex 32`                                   |
| `COACH_AGENT_PRIVATE_KEY`                   | `sui keytool generate ed25519`                           |
| `EXECUTOR_AGENT_PRIVATE_KEY`                | `sui keytool generate ed25519`                           |
| `SETTLEMENT_KEEPER_PRIVATE_KEY`             | `sui keytool generate ed25519`                           |
| `MEMWAL_DELEGATE_ENCRYPTION_KEY`            | `openssl rand -hex 32`                                   |
| `ATOMASDK_BEARER_AUTH`                      | [Atoma dashboard](https://atoma.network)                 |
| `ENOKI_PRIVATE_KEY`                         | [Enoki dashboard](https://docs.enoki.mystenlabs.com)     |
| `GOOGLE_CLIENT_ID` + `GOOGLE_CLIENT_SECRET` | [Google Cloud Console](https://console.cloud.google.com) |
| `TELEGRAM_BOT_TOKEN`                        | [@BotFather](https://t.me/BotFather)                     |
| `TG_USER_ID_PEPPER`                         | `openssl rand -hex 32`                                   |
| `TELEGRAM_WEBHOOK_SECRET_TOKEN`             | `openssl rand -hex 32` (production only)                 |

<Callout variant="danger">
  `JWT_SECRET`, `ENOKI_PRIVATE_KEY`, `TELEGRAM_BOT_TOKEN`,
  `GOOGLE_CLIENT_SECRET`, and all three private keys are server-only. Never
  prefix them with `VITE_` or let them ship to the browser.
</Callout>

<ProseH2 id="web-environment">Web Environment</ProseH2>

Copy `web/.env.example` to `web/.env`. Only two variables are needed:

<CodeBlock
  lang="bash"
  code={`# LIGHTHOUSE WEB .env
# Only VITE_ prefixed vars are exposed to the browser.

VITE_API_BASE_URL=http://localhost:3700
VITE_TELEGRAM_BOT_URL=https://t.me/LighthouseCoachBot`}
/>

<ProseH2 id="database">Database</ProseH2>

Apply the Prisma schema to your PostgreSQL instance:

<CodeBlock
  lang="bash"
  code={`cd backend
bun run db:push`}
/>

<ProseH2 id="optional-relayer">Optional Messaging Relayer</ProseH2>

Sui Stack Messaging is optional for the core coaching flow. To run the relayer locally:

<CodeBlock
  lang="bash"
  code={`git clone https://github.com/MystenLabs/sui-stack-messaging.git ~/sui-stack-messaging
cd ~/sui-stack-messaging/relayer
cp .env.example .env
docker compose up -d`}
/>

<ProseH2 id="run">Run</ProseH2>

<CodeBlock
  lang="bash"
  code={`# Terminal 1: backend on port 3700
cd backend && bun run dev

# Terminal 2: web on port 3200

cd web && bun run dev`}
/>

Open [http://localhost:3200](http://localhost:3200) for the landing page, or message [@LighthouseCoachBot](https://t.me/LighthouseCoachBot) for the live testnet bot.
